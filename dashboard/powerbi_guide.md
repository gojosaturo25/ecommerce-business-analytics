# Power BI Dashboard — Build Guide

This guide walks you through building the Power BI dashboard from the pre-aggregated CSV files generated by the ETL pipeline.

---

## Prerequisites

1. **Power BI Desktop** (free): [Download here](https://powerbi.microsoft.com/desktop)
2. **Data files** in `dashboard/data/`:
   - `monthly_kpis.csv` — Monthly GMV, orders, AOV, on-time rate
   - `category_breakdown.csv` — Revenue and metrics by product category
   - `regional_delivery.csv` — Delivery stats by customer state
   - `seller_leaderboard.csv` — Top 50 sellers by revenue

---

## Step 1: Import Data

1. Open Power BI Desktop
2. Click **Get Data → Text/CSV**
3. Import each CSV file from `dashboard/data/`
4. In the Navigator, click **Load** (not Transform)
5. Repeat for all 4 files

---

## Step 2: Data Relationships

Power BI should auto-detect relationships. Verify:
- No cross-table relationships needed (each CSV is self-contained)
- Ensure `year_month` is sorted chronologically

**Optional:** Add a Date table for time intelligence:
- **Modeling → New Table:**
  ```DAX
  DateTable = CALENDAR(DATE(2016,9,1), DATE(2018,10,1))
  ```

---

## Step 3: Dashboard Pages

### Page 1: Executive Overview

| Element | Type | Data Source | Field(s) |
|---------|------|------------|----------|
| Total GMV | KPI Card | `monthly_kpis` | `SUM(total_gmv)` |
| Total Orders | KPI Card | `monthly_kpis` | `SUM(total_orders)` |
| Avg AOV | KPI Card | `monthly_kpis` | `AVERAGE(avg_order_value)` |
| On-Time Rate | KPI Card | `monthly_kpis` | `AVERAGE(on_time_rate)` formatted as % |
| GMV Trend | Line Chart | `monthly_kpis` | X: `year_month`, Y: `total_gmv` |
| Order Trend | Bar Chart | `monthly_kpis` | X: `year_month`, Y: `total_orders` |
| Avg Review Trend | Line Chart | `monthly_kpis` | X: `year_month`, Y: `avg_review_score` |

**Formatting Tips:**
- Use a **dark theme** for professional look (View → Themes → choose a dark theme)
- Format GMV cards with "R$" prefix and "M" suffix (divide by 1M in DAX)
- Use conditional formatting on On-Time Rate (green >90%, yellow 80-90%, red <80%)

### Page 2: Category Performance

| Element | Type | Data Source | Field(s) |
|---------|------|------------|----------|
| Revenue by Category | Horizontal Bar | `category_breakdown` | Category, Revenue |
| Category Late Rate | Table/Matrix | `category_breakdown` | Category, late_rate |
| Avg Review by Category | Heat Matrix | `category_breakdown` | Category, avg_review |
| Category Filter | Slicer | `category_breakdown` | product_category |

**DAX Measures:**
```DAX
Revenue Share = 
    DIVIDE(
        SUM(category_breakdown[revenue]),
        CALCULATE(SUM(category_breakdown[revenue]), ALL(category_breakdown))
    )
```

### Page 3: Delivery & Regional

| Element | Type | Data Source | Field(s) |
|---------|------|------------|----------|
| Delivery by State Map | Map | `regional_delivery` | customer_state (Location), avg_delivery_days (Value) |
| Late Rate by State | Stacked Bar | `regional_delivery` | customer_state, late_rate |
| State Filter | Slicer | `regional_delivery` | customer_state |
| Delivery Days Distribution | Column Chart | `regional_delivery` | States sorted by avg_delivery_days |

### Page 4: Seller Leaderboard

| Element | Type | Data Source | Field(s) |
|---------|------|------------|----------|
| Seller Table | Table | `seller_leaderboard` | seller_id, seller_state, total_revenue, avg_review_score, total_items_sold |
| Revenue by Seller State | Pie/Donut | `seller_leaderboard` | seller_state, total_revenue |
| Review vs Revenue | Scatter | `seller_leaderboard` | X: total_revenue, Y: avg_review_score |

---

## Step 4: Formatting & Polish

1. **Theme:** Apply a consistent color theme (dark blue/teal works well)
2. **Title:** Add a text box header: "E-Commerce Business Dashboard"
3. **Logo:** Add your name/brand in the corner
4. **Tooltips:** Enable tooltips on all visuals
5. **Mobile layout:** Create a mobile-optimized version (View → Mobile Layout)

---

## Step 5: Export & Share

1. **Save as:** `dashboard/ecommerce_dashboard.pbix`
2. **Screenshots:** Use Snipping Tool to capture each page → save to `dashboard/screenshots/`
3. **PDF Export:** File → Export → PDF for sharing

---

## Alternative: Tableau Public

If using Tableau Public instead:
1. Import the same 4 CSV files
2. Create similar dashboard pages
3. Publish to Tableau Public (free) for an interactive web link
4. Include the link in your README
